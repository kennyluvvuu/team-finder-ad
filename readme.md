# TeamFinder (Вариант #3 — Необходимые навыки проектов и фильтрация)

## Сейчас проект находится в production на хостинге: [teamfind.duckdns.org](https://teamfind.duckdns.org)
**Предлагаю посетить проект, зарегистрироваться и начать работу над Pet-проектами!))**

**TeamFinder** — это современная веб-платформа для разработчиков, дизайнеров и других специалистов, позволяющая находить единомышленников для совместной работы над Pet-проектами. Зарегистрированные пользователи могут публиковать идеи проектов, собирать команду и откликаться на предложения.

<img width="1506" height="946" alt="image" src="https://github.com/user-attachments/assets/10f6396e-b5df-447f-94fb-23a29023d352" />


В соответствии с выбранным **Вариантом №3**, основной фокус реализации направлен на:
* **Необходимые навыки у проектов:** Отображение блока «Необходимые навыки» на детальной странице проекта, возможность добавления и удаления навыков автором проекта без перезагрузки страницы.
* **Фильтрация проектов по навыкам:** Визуальный фильтр тегов на главной странице (списке проектов), позволяющий быстро находить проекты, требующие конкретных технологических стеков, с поддержкой сброса и подсветки активных фильтров.

## Архитектура и важные отклонения от ТЗ

В процессе разработки архитектура проекта была улучшена по сравнению со стандартным монолитным подходом Django, описанным в ТЗ. Ниже приведены ключевые отклонения и их обоснование:

1. **Разделение Frontend и Backend (Decoupled Single Page Application):**
   * **Вместо** использования стандартных HTML-шаблонов Django (`templates_var3/`) и Vanilla CSS/JS (`static/js/skills.js`), проект реализован в виде независимых приложений.
   * **Backend:** Безголовый (headless) API на **Django 5.2+** и **Django REST Framework (DRF)**. Он отдаёт и принимает структурированный JSON, а также автоматически документируется с помощью OpenAPI (Swagger UI доступен по адресу `/api/docs/`).
   * **Frontend:** Современное интерактивное SPA на **React 19** + **TypeScript** + **Vite** + **Tailwind CSS**. Вся логика работы с сервером (включая работу с кэшем, мутациями и инвалидацией данных при обновлении навыков) построена на **TanStack React Query**.
   * *Примечание:* Папки `templates_var1`, `templates_var2`, `templates_var3` и `static` сохранены в репозитории для совместимости, но **не участвуют** в работе запущенного приложения.

2. **Использование Bun как JS-окружения:**
   * Управление зависимостями фронтенда, сборка и серверный запуск происходят через быстрый рантайм **Bun** (альтернатива Node.js/npm). В production-режиме Bun компилирует статические файлы и раздаёт их через легковесный встроенный веб-сервер.

3. **Обратный прокси-сервер (Caddy) в Production:**
   * Для объединения фронтенда и бэкенда на одном домене (что решает проблемы CORS и обмена Cookie сессий) используется веб-сервер **Caddy v2**. Он автоматически проксирует запросы:
     * `/api/*` и `/media/*` ➡️ контейнеру `backend` (Django REST Framework)
     * Все остальные запросы (HTML, JS, статика) ➡️ контейнеру `frontend` (Bun SPA)
   * Caddy также обеспечивает легкую настройку SSL/HTTPS с помощью Let's Encrypt в один клик.

4. **Единый конфигурационный файл `.env`:**
   * Настройки базы данных, ключи безопасности Django, порты и домены настраиваются в одном месте — в файле `.env` в корне проекта. Он автоматически прокидывается во все контейнеры Docker Compose.

## Переменные окружения (`.env`)

Скопируйте пример конфигурации в файл `.env`:
```bash
cp .env.example .env
```

После этого отредактируйте параметры. Ниже приведено описание ключевых переменных:

| Переменная | Описание / Значение по умолчанию |
|---|---|
| `DJANGO_SECRET_KEY` | Секретный ключ Django. Сгенерируйте уникальный для продакшена. |
| `DJANGO_DEBUG` | `True` для разработки, `False` для продакшена. |
| `ALLOWED_HOSTS` | Список разрешенных хостов через запятую (например, `localhost,127.0.0.1,backend,web`). |
| `CSRF_TRUSTED_ORIGINS` | Источники, которым разрешено отправлять unsafe запросы (настроен на порты 3000 и 80). |
| `TASK_VERSION` | Должно быть равно `3` для активации функционала Варианта №3. |
| `POSTGRES_DB` | Имя базы данных PostgreSQL (по умолчанию `team_finder`). |
| `POSTGRES_USER` | Пользователь базы данных (по умолчанию `team_finder`). |
| `POSTGRES_PASSWORD` | Пароль пользователя БД. |
| `POSTGRES_HOST` | Хост СУБД. Укажите `localhost` для запуска на хосте или `db` для Docker Compose. |
| `POSTGRES_PORT` | Порт PostgreSQL (по умолчанию `5432`). |
| `BACKEND_URL` | URL бэкенда для фронтенд-сервера (в Docker Compose: `http://backend:8000`). |
| `SITE_DOMAIN` | Домен для Caddy (например, `localhost` или `teamfinder.local`). |

## Инструкция по запуску в режиме разработки (Development Mode)

В режиме разработки база данных запускается в Docker-контейнере, чтобы не устанавливать PostgreSQL локально, а сам Django-бэкенд и React-фронтенд запускаются непосредственно на вашем компьютере с поддержкой горячей перезагрузки (Hot Reload).

### Шаг 1. Запуск базы данных
Запустите изолированный контейнер базы данных:
```bash
docker compose -f docker-compose.dev.yml up -d
```
База данных станет доступна по адресу `localhost:5432`. Убедитесь, что в `.env` параметр `POSTGRES_HOST=localhost`.

### Шаг 2. Запуск Django Backend
1. **Создайте и активируйте виртуальное окружение Python:**
   * **Linux/macOS:**
     ```bash
     python3 -m venv .venv
     source .venv/bin/activate
     ```
   * **Windows:**
     ```bash
     python -m venv .venv
     .venv\Scripts\activate
     ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```
   *(Или используйте `uv pip install -r requirements.txt` для мгновенной установки)*

3. **Примените миграции базы данных:**
   ```bash
   python manage.py migrate
   ```

4. **Загрузите начальные данные (теги навыков):**
   ```bash
   python manage.py loaddata apps/projects/fixtures/skills.json
   ```

5. **Создайте суперпользователя для доступа к админ-панели:**
   ```bash
   python manage.py createsuperuser
   ```

6. **Запустите локальный сервер Django:**
   ```bash
   python manage.py runserver
   ```
   Бэкенд будет доступен по адресу [http://127.0.0.1:8000](http://127.0.0.1:8000).  
   Swagger UI документация к API доступна по адресу [http://127.0.0.1:8000/api/docs/](http://127.0.0.1:8000/api/docs/).

### Шаг 3. Запуск React Frontend
Для запуска вам понадобится установленный рантайм [Bun](https://bun.sh/).

1. **Перейдите в папку фронтенда:**
   ```bash
   cd frontend
   ```

2. **Установите зависимости:**
   ```bash
   bun install
   ```

3. **Запустите сервер разработки Vite:**
   ```bash
   bun run dev
   ```
   Фронтенд автоматически запустится по адресу [http://localhost:3000](http://localhost:3000).  
   Запросы на `/api` и `/media` будут проксироваться через Vite-сервер на локальный бэкенд `http://127.0.0.1:8000`.

## Инструкция по запуску в режиме продакшена (Production Mode)

В продакшен-режиме всё приложение запускается одной командой в Docker Compose. Сетевое взаимодействие между СУБД, бэкендом, фронтендом и прокси полностью изолировано и защищено.

### Шаг 1. Настройка окружения
Убедитесь, что в файле `.env` указаны следующие значения для работы внутри Docker сети:
```ini
POSTGRES_HOST=db
POSTGRES_PORT=5432
DJANGO_DEBUG=False
BACKEND_URL=http://backend:8000
SITE_DOMAIN=localhost # или ваш публичный домен
```

### Шаг 2. Запуск контейнеров
Запустите сборку и старт всех сервисов в фоновом режиме:
```bash
docker compose up --build -d
```
Эта команда поднимет 4 контейнера:
1. `teamfinder_db` (Postgres 16) — сохраняет данные в volume `postgres_data`.
2. `teamfinder_backend` (Django REST Framework) — автоматически применит миграции при запуске.
3. `teamfinder_frontend` (Bun React App) — соберет продакшен-билд (`vite build`) и запустит веб-сервер.
4. `teamfinder_caddy` (Caddy Reverse Proxy) — прослушивает порты `80` и `443`.

### Шаг 3. Загрузка данных и создание суперпользователя в Docker
Поскольку база данных создана с нуля в контейнере, вам необходимо загрузить навыки и создать администратора внутри контейнера бэкенда:

1. **Загрузите фикстуры навыков:**
   ```bash
   docker exec -it teamfinder_backend python manage.py loaddata apps/projects/fixtures/skills.json
   ```

2. **Создайте суперпользователя:**
   ```bash
   docker exec -it teamfinder_backend python manage.py createsuperuser
   ```

### Шаг 4. Доступ к приложению
Перейдите по адресу [http://localhost](http://localhost) (или по вашему адресу `SITE_DOMAIN`).  
* Caddy направит вас на React SPA.
* Панель администратора Django доступна по адресу [http://localhost/admin/](http://localhost/admin/).
* Swagger API документация доступна по адресу [http://localhost/api/docs/](http://localhost/api/docs/).

Чтобы остановить продакшен-окружение, выполните:
```bash
docker compose down
```

## Тестирование функционала Варианта №3

Для ручной проверки логики проекта выполните следующие действия:

1. **Регистрация пользователей:**
   * Перейдите на `/register`, создайте двух пользователей (например, `Иван Иванов` и `Петр Петров`).
2. **Создание проекта:**
   * Войдите под аккаунтом `Иван Иванов` и создайте проект (кнопка **«Создать проект»** на главной странице).
3. **Добавление необходимых навыков к проекту:**
   * Откройте страницу созданного проекта. Будучи его автором, вы увидите кнопку **«+ Добавить навык»**.
   * Введите название навыка (например, `Python`). Появится выпадающий список автодополнения (поиск идет по базе данных). Выберите `Python`.
   * Попробуйте ввести навык, которого нет в базе (например, `WebSockets`). Появится опция **Создать "WebSockets"**. Кликните по ней. Новый навык запишется в базу данных и мгновенно привяжется к проекту без перезагрузки страницы.
4. **Удаление навыков:**
   * Нажмите на крестик `(x)` рядом с любым тегом на странице проекта. Навык будет отвязан от проекта в реальном времени.
5. **Фильтрация на главной странице:**
   * Перейдите на страницу со списком проектов. В верхней части экрана вы увидите панель **«Фильтр по необходимым навыкам»**, где выводятся все используемые в проектах навыки по алфавиту.
   * Кликните по навыку `Python`. Список проектов отфильтруется, останутся только проекты с этим навыком, а в URL добавится параметр `?skill=Python`. Активный навык подсветится.
   * Нажмите на кнопку **«Сбросить»** рядом с активным фильтром, чтобы вернуть весь список проектов.
6. **Права доступа:**
   * Выйдите из аккаунта Ивана и зайдите под Петром Петром (или просматривайте проект анонимно). На странице проекта Ивана блок навыков отобразится **без** кнопок добавления и крестиков удаления (только просмотр).
